## 코드 블록 언어 표시

`docs/` 하위 마크다운 파일에서 코드 예시를 작성할 때는 언어 태그를 **`markdown`** 으로 표시한다.

```
\```markdown
// 코드 예시
\```
```

**이유:** IDE에서 `java` 등 언어명 지정 시 빨간 줄(lint)이 떠서 불편함.
**주의:** `java`, `python` 등 실제 언어명으로 바꾸지 말 것.

## 학습 노트 정리 필수 항목

`docs/` 하위 파일에 개념을 정리할 때 아래 항목을 반드시 포함한다.

| 항목 | 설명 |
|------|------|
| 왜 쓰는지 | 이 개념/기능이 존재하는 이유, 해결하는 문제 |
| 언제 쓰는지 | 적합한 사용 상황, 조건 |
| 장점 | 사용했을 때의 이점 |
| 단점 | 한계, 트레이드오프 |
| 특징 | 다른 것과 구별되는 핵심 성질 |
| 주의할 점 | 실수하기 쉬운 부분, 안티패턴 |

모든 항목이 항상 별도 섹션일 필요는 없다. 테이블, 인라인 설명 등 형식은 자유롭게 사용하되 내용은 빠짐없이 담는다.
