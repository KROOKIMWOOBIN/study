## 동기 VS 비동기

### 동기
- 요청한 쪽이 결과를 직접 받아야 다음 흐름으로 진행한다.

### 비동기
- 요청만 하고, 작업 완료 통지는 콜백/이벤트 등으로 전달된다.

### 블로킹
- 그 자리에서 아무것도 못 함
- 작업이 끝날 때까지 현재 스레드가 멈춰 있음

### 논블로킹
- 다른 일 가능
- 작업이 끝나지 않았더라도 즉시 제어권을 돌려받음

### 비유
- 식당에서 음식 주문하기
  - 손님 = 호출자(스레드)
  - 주방 = 작업 수행자
  - 음식 = 결과

|상태| 상황                                                             | 특징                                            |
|--|----------------------------------------------------------------|-----------------------------------------------|
|동기 + 블로킹| 1. 카운터 앞에서 주문<br>2. 음식 나올 때까지 그 자리에서 서서 기다림<br>3. 음식 받으면 다음 행동 | - 결과를 직접 기다림 → 동기<br>- 기다리는 동안 아무것도 못 함 → 블로킹 |
|동기 + 논블로킹|1. 주문 후 주방 창구를 계속 힐끔힐끔 봄<br>2. 아직 안 나오면 옆에서 서성이다가 다시 확인<br>3. 결국 내가 직접 받음| - 결과는 내가 직접 확인 → 동기<br>- 기다리는 동안 움직일 수 있음 → 논블로킹|
|비동기 + 블로킹|1. 진동벨을 받음 (비동기 구조)<br>2. 하지만 테이블에 가만히 앉아서 아무것도 안 함<br>3. 벨 울리면 음식 받음| - 결과는 알림으로 받음 → 비동기<br> - 스스로 행동은 멈춤 → 블로킹|
|비동기 + 논블로킹|1. 진동벨 받음<br> 2. 자리 가서 이야기하고, 휴대폰 보고, 다른 일 함<br> 3. 벨 울리면 음식 받음| - 결과는 알림 → 비동기<br>- 기다리는 동안 자유 → 논블로킹|
