# Java 기초 — 객체 지향 4원칙

객체 지향 프로그래밍(OOP)의 4가지 핵심 원칙. 좋은 코드 설계의 기반이 된다.

---

## [캡슐화](./캡슐화.md)

객체의 내부 상태와 구현을 외부로부터 숨기고, 메서드를 통해서만 접근하게 한다.

| 개념 | 설명 |
|------|------|
| 캡슐화 | 상태를 메서드로만 조작, 변경 지점을 클래스 내부로 한정 |
| 접근 제어자 | `private` / `default` / `protected` / `public` |

---

## [다형성](./다형성.md)

부모 타입 하나로 서로 다른 자식 객체를 동일하게 다루는 것.

| 개념 | 설명 |
|------|------|
| 업캐스팅 | 자식 → 부모 타입 (자동 형변환) |
| 다운캐스팅 | 부모 → 자식 타입 (명시적, `ClassCastException` 주의) |
| 동적 바인딩 | 실행 시점에 실제 객체 기준으로 메서드 결정 |

---

## [추상화](./추상화.md)

구현이 아닌 역할 중심으로 설계하여 변경에 강하고 확장에 유연한 구조를 만든다.

| 개념 | 설명 |
|------|------|
| 인터페이스 vs 추상 클래스 | 역할 계약 vs 역할 + 구현 |
| is-a / has-a | 상속 관계 vs 합성 관계 |

---

## [상속](./상속.md)

공통 개념을 상위 타입으로 묶어 다형성 기반을 제공한다.
상속은 코드 재사용보다 **is-a 관계 표현**이 목적이다.

| 개념 | 설명 |
|------|------|
| 상속의 위험성 | 강한 결합, 캡슐화 붕괴, 부모 변경이 자식에 전파 |
| 다중 상속 | 클래스는 단일 상속, 인터페이스는 다중 구현 가능 |
