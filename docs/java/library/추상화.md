## 추상화(Abstract, Interface)
### 목적
구현이 아닌, 역할 중심으로 설계를 하여 변경에 강하고, 확장에 유연한 구조를 만들기 위해 (OCP원칙)
### 기능
1. 코드 확장성이 좋아진다.
2. 구현을 숨기고 역할만 바라보게 설계가 가능하다.
3. 한 타입으로 여러 객체를 다룰 수 있어 코드가 단순해 진다.
4. 중복 코드 제거
### 주의사항
- 클래스는 다중 상속이 불가능하지만 인터페이스는 가능하다.
  - 클래스는 다이아몬드 구조 때문에 A, B 중 어떤 메서드를 호출해야 하는 지 모르기 때문이다.
  - 인터페이스는 자식이 구현하기 때문에 다이아몬드 구조 문제가 발생하지 않는다.
- 자바8 이후부터는 인터페이스에 default method 가 생겼다. 
  - default method -> Override 가 가능하다.
  - default method 때문에 인터페이스도 다이아 구조가 생겼다.

### is-a
- A is a B
- A는 B의 한 종류이다.
- K9는 자동차의 한 종류이다.

### has-a
- A has a B
- A는 B를 가지고 있다.
- 자동차는 바퀴를 가지고 있다.
