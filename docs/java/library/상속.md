## 상속
- 상속의 목적은 코드 재사용이 아니라 공통 개념 표현과 다형성 기반 확보이다.

### 공통 개념
- 여러 클래스가 동일한 본질(is-a) 을 가질 때 상위 타입으로 묶어 의미를 드러냄
```java
class Dog extends Animal { }
class Cat extends Animal { }
```

### 다형성 기반 제공
- 부모 타입으로 자식 객체를 다룰 수 있음
- 동적 바인딩을 통한 행위 위임
```markdown
Animal animal = new Dog();
animal.sound(); // 멍멍 출력
```

## 상속의 위험성
- 상속은 가장 강한 결합을 만든다.

### 문제점
1. 부모 변경이 자식에 전파됨
   - 부모 수정 → 모든 자식 영향
   - 사이드 이펙트 발생 가능성 큼
2. 잘못된 is-a 관계
   - 개념적으로 성립하지 않는 상속
   ```java
   class Engine extends Car {} // is-a가 아닌 has-a 관계
   ```
3. 캡슐화 붕괴
   - 부모의 내부 구현에 자식이 의존
   - 부모 구현 변경 시 자식 로직 깨짐
4. 유연성 부족
   - 런타임에 상속 관계 변경 불가
   - 조합이 어려움

## 다중 상속
- 하나의 클래스가 둘 이상의 부모 클래스를 상속받는 것
- 자바에서는 구현 클래스는 1개만 제한하며, 역할 상속(Interface)은 제한이 없다. 이유는 다이아몬드 구조 이슈
- 자바8 이상부터는 `Interface`에 `default Method`가 생기면서 다이아몬드 이슈가 다시 발생하였다.
```java
class Dog extends Animal implements Move {} 
```
