## 상속

### 정의
기존 클래스의 속성과 기능을 재사용하여 새로운 클래스를 만드는 것

### 목적
1. 중복 제거
2. 공통 로직의 재사용
3. 계층 구조 표현
```java
class Animal {
    void sound() {}
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("멍멍");
    }
}
```

### 주의점(실무 중요)
- 상속은 is-a 관계일 때만 사용
- 잘못 쓰면 결합도 폭증
```java
class Car extends Engine {} // is-a 아님 has-a 관계임
```

### 다중 상속
- 하나의 클래스가 둘 이상의 부모 클래스를 상속받는 것
- 자바에서는 구현 클래스는 1개만 제한하며, 역할 상속은 제한이 없다. 이유는 다이아몬드 구조 이슈
- 자바8 이상부터는 Interface에 default Method가 생기면서 다이아몬드 이슈가 다시 발생하였다.
```java
class Dog extends Animal implements Move {} 
```
