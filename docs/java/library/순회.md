## 순회
- 자료 구조마다 데이터를 접근하는 방법이 다 달라 접근을 할려면 내부 구조를 알았어야 했다. 그래서 모든 자료 구조를 일관성 있는 방법으로 접근하기 위해 아래에 인터페이스를 제공한다.

## Iterable
- [반복 가능한]이라는 뜻이다.
- “나는 반복될 수 있다”는 능력 선언

### 주요 메서드
```java
public interface Iterable<T> {
    Iterator<T> iterator();
}
```
- 단순한 Iterator 반복자를 반환한다.

## Iterator
- [반복자]라는 뜻이다.
- 실제로 요소를 하나씩 꺼내는 도구

### 주요 메서드
```java
public interface Iterator<E> {
    boolean hasNext();
    E next();
}
```
| 이름        | 뜻                                         |
|-----------|-------------------------------------------|
| hasNext() | 다음 요소가 있는 지 확인한다. 다음 요소가 없으면 false를 반환한다. |
| next()    | 다음 요소를 반환한다. 내부에 있는 위치를 다음으로 이동한다.        |

## 향상된 반복문 (for-each)
- 반복 가능하고 반복자가 구현된 [컬렉션 or 배열]는 향상된 반복문을 사용할 수 있다.
```java
public class Main {
    public static void main(String[] args) {
        int[] numberArr = new int[]{1, 2, 3, 4};
        for (int number : numberArr) {
            System.out.println("number = " + number);
        }
    }
}
```
- 컬렉션일 경우 실제 구조
```java
public class Main {
    public static void main(String[] args) {
        int[] numberArr = new int[]{1, 2, 3, 4};
        MyArray myArray = new MyArray(numberArr);
        for (Iterator<Integer> it = myArray.iterator(); it.hasNext();) {
            Integer i = it.next();
        }
    }
}
```
