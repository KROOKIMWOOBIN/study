## 배열
### 배열의 특징
1. 배열에서 자료를 찾을 때 인덱스를 사용하면 매우 빠르게 자료를 찾을 수 있다.
2. 인덱스를 통한 입력, 변경, 조회의 경우 한번의 계산으로 자료의 위치를 찾을 수 있다.
### 배열의 인덱스 조회 공식
- 배열의 시작 참조 + (자료의 크기 * 인덱스 위치)
### 배열의 검색
- 배열의 크기가 클수록 오랜 시간이 걸린다.
- 배열의 순차 검색은 배열에 들어있는 데이터의 크기 연산이 필요한다. 배열의 크기가 n이면 연산도 n만큼 필요하다.
### 배열의 추가
- 처음 위치 추가 O(n)
- 중간 위치 추가 O(n/2)
- 마지막 위치 추가 O(1)
### 배열의 단점
1. 배열의 길이를 동적으로 변경할 수 없다.
2. 데이터를 추가하기 불편하다.
## 빅오(O) 표기법
### 빅오 표기법 예시
- O(1) - 상수 시간
  - 입력 데이터의 크기에 관계없이 알고리즘의 실행 시간이 일정한다.
  - 예) 배열에서 인덱스를 사용하는 경우
- O(n) - 선형 시간
  - 알고리즘 실행 시간이 입력 데이터의 크기에 비례하여 증가한다.
  - 예) 배열의 검색, 배열의 모든 요소를 순회하는 경우
- O(n*n) - 제곱 시간
  - 알고리즘 실행 시간이 입력 데이터의 크기의 제곱에 비례하여 증가한다.
  - 예) 이중 루프를 사용하는 알고리즘에서 나타남
- O(log n) - 로그 시간
  - 알고리즘의 실행 시간이 데이터 크기의 비례하여 증가한다.
  - 예) 이진 탐색
- O(n log n) - 선형 로그 시간
  - 예) 많은 효율적인 정렬 알고리즘들
### 상황 가정
- 최적의 경우
  - O(1)
- 최악의 경우
  - O(n)
- 평균의 경우
  - O(n/2)
## 리스트
### 설명
순서가 있고 중복을 허용하는 자료 구조를 리스트라 한다.

### 배열 리스트(ArrayList)
#### 단점
- 메모리를 낭비할 수 있다.
- 앞이나 중간에 [추가,삭제]를 할 경우 O(n)이므로 성능이 좋지 않다.
#### 배열 리스트의 빅오
- 데이터 추가
  - 마지막에 추가: O(1)
  - 앞, 중간에 추가: O(n)
- 데이터 삭제
  - 마지막에 삭제: O(1)
  - 앞, 중간에 삭제: O(n)
- 인덱스 조회: O(1)
- 데이터 검색: O(n)
