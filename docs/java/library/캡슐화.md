## 캡슐화
### 목적 
내부는 숨기고, 필요한 것만 공개해서 시스템을 안정적으로 유지하는 것
### 기능
1. 데이터 보호(무분별한 수정 방지)
2. 객체 상태를 일관되게 유지
3. 내부 구현을 숨겨서 변경에 강한 구조
4. 보안, 안정성 증가

### 접근제어자
1. public
   - 모든 패키지에서만 접근이 가능하다.
2. default
   - 같은 패키지에서만 접근이 가능하다.
3. protected
   - 같은 패키지 또는 다른 패키지에서 상속받은 클래스만 접근이 가능하다.
4. private
   - 같은 클래스에서만 접근이 가능하다.

### 예시
```java
public class Main {
  public static void main(String[] args) {
    Time time = new Time();

    time.setHour(25); // 유효하지 않은 파라미터
    System.out.println("Hour: " + time.getHour());

    time.setHour(13); // 유효한 파라미터
    System.out.println("Hour: " + time.getHour());
  }
}
class Time {
    // 시간을 외부에서 접근할 수 없도록 private로 선언한다.
    private int hour;
    public void setHour(int hour) {
        // 시간에 대한 유효성 검사
        if(hour < 0 || hour > 24) {
            return;
        }
        this.hour = hour;
    }
    public int getHour() {
        return hour;
    }
}
```
