## 멀티태스킹 VS 멀티프로세싱
- 멀티프로세싱은 하드웨어 장비의 관점이고, 멀티태스킹은 운영체제 소프트웨어의 관점이다.

### 멀티태스킹
- 단일 CPU 코어가 여러 작업을 동시에 수행하는 것처럼 보이게 하는 것을 의미한다. 
- 소프트웨어 기반으로 CPU 시간을 분할(시분할)하여 각 작업에 할당한다.

### 멀티프로세싱
- 여러 CPU 코어를 사용하여 동시에 여러 작업을 수행하는 것을 의미한다. 
- 하드웨어 기반으로 성능을 향상시킨다.

## 프로세스
- 프로세스는 실행 중인 프로그램의 인스턴스이다. 각 프로세스는 독립적인 메모리 공간을 가지며, 운영체제에서 독립된 실행 단위로 취급된다.

### 설명
- 각 프로세스는 독립적인 메모리 공간을 갖고 있으며, 운영체제에서 별도의 작업 단위로 분리해서 관리된다.
- 특정 프로세스(프로그램)에 심각한 문제 가 발생하면 해당 프로세스만 종료되고, 다른 프로세스에 영향을 주지 않는다.

### 구성
1. 코드 섹션: 실행할 프로그램의 코드가 저장되는 부분
2. 데이터 섹션: 전역 변수 및 정적 변수가 저장되는 부분
3. 힙 (Heap): 동적으로 할당되는 메모리 영역
4. 스택 (Stack): 메서드 호출 시 생성되는 지역 변수와 반환 주소가 저장되는 영역

## 스레드
- 스레드는 프로세스 내에서 실행되는 작은 단위이다. 여러 스레드는 하나의 프로세스 내에서 자원을 공유하며, 프로세스의 코드, 데이터, 시스템 자원등을 공유한다.

### 메모리 구성
- 공유 메모리
  - 같은 프로세스의 코드 섹션, 데이터 섹션, 힙은 프로세스 안의 모든 스레드가 공유한다.
- 개별 스택
  - 각 스레드는 자신의 스택을 갖고 있다.

### 설명
- 단일 스레드
  - 한 프로세스 내에 하나의 스레드만 존재
- 멀티 스레드 
  - 한 프로세스 내에 여러 스레드가 존재

## 스케줄링
- 여러 개의 스레드 중에서 지금 이 순간 CPU를 사용할 스레드를 선택하는 과정이다.

### 스케줄링이 필요한 이유
- CPU 코어 수보다 실행 가능한 스레드 수가 더 많기 때문이다.
- 따라서 모든 스레드를 동시에 실행할 수 없고, 실행 순서를 정해 번갈아가며 실행해야 한다.

## 컨텍스트 스위칭
- CPU를 사용하던 스레드의 실행을 멈추고, 다른 스레드로 실행을 전환하는 과정이다.

### 컨텍스트 스위칭이 필요한 이유
- 스케줄링에 의해 실행할 스레드가 바뀌기 때문이다.
- 이전 스레드는 나중에 다시 실행될 수 있으므로, 다양한 정보를 저장해야 한다.
- 이후 다시 실행될 때 이 정보를 불러와, 중단된 지점부터 계속 실행할 수 있다.
