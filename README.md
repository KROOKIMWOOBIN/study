## [정리 방법]
1. 왜 쓰는지 : 이 개념/기능이 존재하는 이유, 해결하는 문제
2. 어떻게 쓰는지 : 실제 사용 방법, 코드 예시, 설정 방법
3. 언제 쓰는지 : 적합한 사용 상황, 조건
4. 장점 : 사용했을 때의 이점
5. 단점 : 한계, 트레이드오프
6. 특징 : 다른 것과 구별되는 핵심 성질
7. 주의할 점 : 실수하기 쉬운 부분, 안티패턴
8. 베스트 프랙티스는?
9. 실무에서는 어떻게 사용?
